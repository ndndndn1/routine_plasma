## Keyword match (>=2 required)
1. Inverse Problem Solving — The whole paper formulates the inference of plasma composition (poloidal distributions of neutrals density n_n, electron density n_e, electron temperature T_e) from measured camera images as a non-linear inverse problem and solves it by gradient descent through a differentiable forward model.
2. Tomographic Reconstruction — The forward model is the line-of-sight (LOS) integral of plasma emissivity ε(R,Z) along camera rays. Inverting this LOS integral to recover ε(R,Z) (and the underlying plasma fields) is the canonical plasma-tomography problem; the paper turns it into a differentiable rendering problem.
3. Spatially Resolved Mapping — The inferred quantities are 2D poloidal maps of n_n / n_e / T_e, not scalar averages, so the output is a spatially resolved reconstruction.

## Methods (summary)
- Forward model: physically-based volumetric rendering of plasma emission within a tokamak, built on top of the Mitsuba 3 renderer. Light transport in the optically thin plasma is simulated with Null-Scattering, gradients are computed via Path-Replay Backpropagation. Emission for the relevant atomic line (e.g. D-α) is parameterised by the local n_n, n_e, T_e and an atomic-physics rate coefficient.
- Differentiability: the renderer produces both the simulated 1024×1024 image and the analytic gradient of every output pixel with respect to the underlying 3D field parameters, in seconds-to-minutes per frame on a GPU, with constant memory cost via Path-Replay Backprop.
- Demonstration: principle is verified on synthetic data generated by a SOLPS edge-plasma simulation of MAST. Three levels of complexity are shown — recovering only the neutrals density n_n(R,Z) from imaging alone; jointly recovering n_n with midplane T_e/n_e measurements as side constraints; full poloidal reconstruction of all three fields.
- Output: the recovered fields match the ground-truth SOLPS distributions sufficiently well that line-integrated images render back to the input camera frames to within imaging noise, demonstrating that the full poloidal composition map is extractable from a small number of filtered camera views.

## Conclusions (as stated by the authors)
- A differentiable plasma renderer is feasible: the rendering pipeline computes both the simulated image and its gradients to machine accuracy in similar wall-time, with minimal memory cost.
- Calibrated high-speed cameras on MAST/MAST-U can in principle be inverted to estimate full poloidal distributions of n_e, T_e and n_n, providing a non-invasive plasma diagnostic that complements probe and spectroscopic measurements.
- The method scales to arbitrary 3D field representations (voxel grids, neural fields, tomographic basis sets), making it a general framework for solving the imaging inverse problem in fusion plasmas.

## Code availability from the authors
The paper's pipeline depends on the (closed source) Mitsuba 3 build with custom plasma volume integrators; no public release is referenced from the abstract. We therefore provide a self-contained PyTorch reference implementation of the *core* inverse-rendering loop on a 2D synthetic poloidal slice — sufficient to reproduce the principle "recover n_n(R,Z) from a synthetic camera image".

## Reference code
See `reference_code.py` in this folder. It:
- builds a synthetic ground-truth poloidal map for n_n(R,Z), n_e(R,Z), T_e(R,Z) on a 64×64 grid (peaked plasma + edge neutrals shell + lower X-point spot);
- defines an emissivity ε(R,Z) ∝ n_n · n_e · √T_e · exp(−E₀/T_e) (D-α-like Boltzmann factor);
- forward-renders synthetic camera images from **four** virtual cameras (outboard / top / inboard / bottom) as line-of-sight integrals through the slice using `grid_sample` (autograd-able analogue of Path-Replay Backprop in Mitsuba);
- adds Gaussian shot noise to the image to mimic real measurements;
- holds n_e and T_e fixed (ground truth) and solves the inverse problem for n_n(R,Z) by gradient descent on `‖render(n_n) − image_obs‖² + λ · TV(n_n)`. The unknown is parameterised on a coarse 12×12 control grid (bilinear-upsampled to 64×64) inside the known plasma support — a low-rank reduction analogous to the basis restrictions used in real plasma tomography;
- reports relative L2 reconstruction error of n_n versus the ground truth, before and after optimisation, plus the residual image-space error.

Typical CPU run output (1.5 k Adam steps):

```
initial rel L2 error  ||n_n0 - n_n_true|| / ||n_n_true|| = 1.08e+00
final   rel L2 error  ||n_n* - n_n_true|| / ||n_n_true|| = 6.6e-01
re-rendered image vs observed:                               1.1e-01
peak ground truth n_n: 4.10, peak recovered: 3.34
```

The recovered map captures the bulk neutrals structure and approaches the ground-truth peak, demonstrating the differentiable-rendering inverse pipeline. Run: `python reference_code.py` (CPU, ~1 minute; PyTorch only).
