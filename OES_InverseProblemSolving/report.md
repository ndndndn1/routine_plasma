## Keyword match (>=2 required)
1. OES (Optical Emission Spectroscopy) — The full diagnostic is built around emission-line intensity ratios of Xe I / Xe II lines from a low-pressure microwave discharge.
2. Inverse Problem Solving — The inverse mapping from a noisy, model-distorted line-ratio vector back to bulk plasma parameters (electron density n_e) is the ill-posed problem the network solves.
3. Surrogate Modeling — The neural network is a learned surrogate that compresses the collisional–radiative (CR) forward model and an instrument-disturbance model into one inverse operator, replacing per-shot non-linear least-squares CR fitting.

## Methods (summary)
- Forward physics model: a state-to-state collisional–radiative model for Xe I that maps (n_e, T_e, gas density, optical thickness) to emission-line intensities of selected Xe I transitions in the visible/NIR; line ratios are used as features to suppress absolute-calibration uncertainty.
- Instrument-disturbance model: a parametric noise / distortion model is superimposed on synthetic line ratios — random multiplicative gain per line (instrument response), additive Gaussian noise, and bounded perturbations of selected cross sections that emulate uncertainty in atomic data.
- Training data: a synthetic database is generated by Monte Carlo sampling over n_e (10^16–10^19 m^-3), T_e, gas density, optical-depth flag, and the instrument-disturbance parameters. Each sample provides (line-ratio vector → n_e).
- Network: feed-forward MLP that takes the log of selected Xe I line ratios (and optionally one Xe II line) as input and outputs normalized log(n_e). The instrument and atomic-data uncertainty are absorbed during training as data augmentation, so at inference time the network is implicitly Bayesian-marginalized over those nuisance parameters.
- Validation: predictions are benchmarked against (i) the canonical line-ratio CR inversion on synthetic test sets and (ii) microwave-interferometry measurements on an experimental Xe microwave discharge.

## Conclusions (as stated by the authors)
- The neural-network OES inversion delivers n_e values that agree with independent microwave-interferometry measurements across the explored discharge conditions while traditional single-ratio CR inversion shows visible bias when instrument noise and cross-section uncertainty are not corrected.
- Training-time data augmentation with the instrument-disturbance model is what allows the network to suppress systematic bias — i.e. the inverse model encodes a learned prior over the nuisance parameters.
- The methodology is generic: any plasma where a CR model and a few well-resolved emission lines are available can re-use the recipe (synthetic CR + noise model + MLP) for non-invasive density diagnostics.

## Code availability from the authors
The publication is closed-access at AIP and we found no companion repository linked from the article landing page. We therefore provide a self-contained reference implementation that reproduces the *structure* of the inverse pipeline: synthetic CR-like forward model + instrument-disturbance augmentation + MLP that infers n_e from line ratios.

## Reference code
See `reference_code.py` in this folder. It:
- builds a toy two-level / three-level CR-like forward model whose line-intensity ratios are monotonic, nonlinear functions of n_e and T_e;
- augments each synthetic sample with multiplicative line-gain noise and additive Gaussian noise to emulate instrument distortion;
- trains an MLP (line-ratio vector → log n_e) with Huber loss;
- evaluates against (a) a clean test set and (b) a corrupted test set with the same disturbance model.

Run: `python reference_code.py` (needs `numpy`, `torch`).
